🧭 Foundations: How the Web Works

Before touching code, let's build a clear mental model of what we're actually making. Spend two hours here and everything that follows will feel far less mysterious.

🎯

By the end of this module you'll understand: what happens when you visit a website, the difference between front-end and back-end, what each language we'll learn is for, and what tools you need installed.

What actually happens when you open a website?

Imagine ordering food at a restaurant. You (the customer) don't go into the kitchen — you tell the waiter what you want, the waiter takes it to the kitchen, and food comes back. The web works the same way.

A few words you'll hear constantly, in plain English:

WordPlain-English meaning
BrowserThe app you view websites in (Chrome, Edge…). It runs the front-end.
ServerA computer, somewhere, that's always on and answers requests.
RequestA message from the browser asking for something ("give me the user list").
ResponseThe server's reply — usually HTML to show, or data (JSON).
URLThe address of a thing on the web, e.g. https://example.com/about.
HTTPThe "language" / rules browsers and servers use to talk. (More in the REST module.)

Front-end vs back-end

This is the single most important distinction in web development. Learn it once and a hundred things make sense.

🎨 Front-end (client-side)

Everything the user sees and touches. Buttons, colours, layout, animations, forms. It runs inside the browser, on the user's device.

Built with: HTML, CSS, JavaScript, and React.

⚙️ Back-end (server-side)

The brain behind the scenes. Checking passwords, calculating totals, deciding who's allowed to do what, and talking to the database. The user never sees it directly.

Built with: Node.js, REST APIs, and a database (MSSQL).

🔑

"Full-stack" just means you can work on both the front-end and the back-end. That's the goal of this whole course — and it's very achievable.

The languages — and what each one is for

People panic at "so many languages." Don't. Each has one clear job. Here's the whole stack on one page:

TechIts one jobAnalogy
HTMLThe structure & content of a pageThe skeleton 🦴
CSSThe styling — colours, spacing, layoutThe skin & clothes 👕
JavaScriptThe behaviour — making things interactiveThe muscles & brain 🧠
ReactA toolkit for building big UIs in JavaScriptPower tools for building 🛠️
Node.jsLets JavaScript run on the serverSame brain, different body 🤖
REST APIThe rules front-end & back-end talk byA waiter taking orders 🧑‍🍳
MSSQLStores data permanentlyA giant, organised filing cabinet 🗄️
😌

Good news: JavaScript is the through-line. Once you know it (Module 1), React, Node and even talking to the database all use the same language. You're really learning one language deeply, then applying it everywhere.

HTML & CSS in five minutes

We'll lean on React for UI, but you must recognise HTML and CSS. Here's the 80% you need.

HTML = content wrapped in tags

An HTML element is an opening tag, some content, and a closing tag:

index.html 
<h1>My first heading</h1>
<p>This is a paragraph of text.</p>
<a href="https://google.com">A link</a>
<button>Click me</button>
<img src="cat.jpg" alt="a cat" />

Tags nest inside each other to build structure — a page is just tags inside tags inside tags, like Russian dolls.

CSS = rules that style those tags

styles.css 
h1 {
  color: navy;        /* property: value; */
  font-size: 32px;
}
button {
  background: dodgerblue;
  border-radius: 8px;
  padding: 10px 20px;
}

Each rule says: "find these elements (the selector) and apply these styles." That's genuinely most of CSS.

💡

You won't write tons of raw HTML/CSS in this course — React generates HTML for you, and we'll style with simple CSS. But knowing these three tags and the rule shape above is enough to follow along.

Your toolkit — what to install

You need surprisingly little. Install these before Day 4 (you won't need them for the JavaScript and React modules, which run right here in the browser).

  1. A code editor — VS Code. Free, from code.visualstudio.com. This is where you'll write code. Install the "Prettier" extension too — it auto-formats your code so it always looks clean.
  2. Node.js (LTS version). From nodejs.org. This lets you run JavaScript outside the browser and gives you npm (the tool that installs code packages). Pick the "LTS" (Long-Term Support) download.
  3. A modern browser — Chrome or Edge. You almost certainly have one. Press F12 to open its "DevTools" — your window into what a page is doing.
  4. SQL Server + SSMS (for the database module, Day 6). We'll cover installing SQL Server Express (free) and SQL Server Management Studio there, so don't worry about it yet.
✅

Verify Node installed correctly. Open a terminal (in VS Code: Terminal → New Terminal) and type node -v then npm -v. If you see version numbers, you're ready.

What's a "terminal"?

It's a text window where you type commands to your computer instead of clicking. It looks intimidating but you'll only use a handful of commands:

CommandWhat it does
node app.jsRun a JavaScript file
npm install expressDownload & add a package to your project
npm run devStart your project in development mode
cd my-folder"Change directory" — move into a folder

The developer mindset (this matters more than syntax)

🔍

Read error messages. They're not insults — they're directions. The red text usually tells you the file and line number. Beginners ignore errors; developers read them first.

🐢

Go small. Build one tiny piece, run it, confirm it works, then add the next. Don't write 100 lines and pray.

🔎

Searching is a skill, not cheating. Every professional Googles things hourly. Learning what to search is the real skill.

💪

Struggle is the curriculum. The moment something doesn't work and you figure out why — that's when learning actually happens.

Quick check

1. A user clicks "Log in". Whose job is it to check whether the password is correct?
The front-end (browser)
The back-end (server)
The CSS file
✅ The back-end. Checking passwords involves secret data and security rules, so it must happen on the server — never in the browser, where users could tamper with it.
2. Which technology's job is to make a page interactive (respond to clicks)?
HTML
CSS
JavaScript
✅ JavaScript adds behaviour. HTML is structure, CSS is styling, JavaScript is the muscle that makes things do stuff.
3. What does a server send back after receiving a request?
A response (HTML or data)
Another request
Nothing — requests are one-way
✅ The request–response cycle: the browser asks (request), the server answers (response). Every page load and button click that fetches data follows this pattern.

Key takeaways

  • The web is a request–response conversation between a browser and a server.
  • Front-end = what users see (browser). Back-end = the logic & data (server). Full-stack = both.
  • HTML = structure, CSS = style, JavaScript = behaviour. React/Node/MSSQL build on top.
  • JavaScript is the one language that runs everywhere in this course — learn it well next.
  • Install VS Code and Node.js; you'll use the terminal for a few simple commands.
