How LLMs Actually Work

An LLM is a fancy autocomplete: predict the next token, billions of times. Understanding that simple loop unlocks the rest of modern AI.

Kid Translation: A Very Smart Autocomplete

🧒 Kid Translation

You know how your phone suggests the next word as you type? "I'll be there in five..." → suggestion: "minutes." That's autocomplete.

An LLM is autocomplete on steroids. It's been trained on basically the whole internet, and its only job is: given a bunch of text, what word comes next?

That sounds boring. But if your autocomplete is good enough, you can ask it: "Q: What is 17 × 23? A:" and it will autocomplete "391." You can ask it: "Translate to French: I am tired →" and it autocompletes "Je suis fatigué." You can paste a coding problem and it autocompletes the solution.

"Predicting the next word" turns out to require understanding language, math, code, history, and reasoning — because the right next word depends on all of those. That's the magic.

So when you "talk" to ChatGPT or Claude, what's actually happening is: the model is doing autocomplete on a transcript that looks like a dialogue. It writes one token, sees the result, and writes the next one — over and over until it's done.

One Token at a Time: Generation

This is called autoregressive generation. "Autoregressive" just means: the model's own previous outputs become its next inputs.

Every single token requires a full pass through all the model's layers — that's hundreds of billions of multiplications per token. This is why generating long responses takes time and costs money.

Sampling: Why the Same Prompt Gives Different Answers

The model doesn't actually output "Paris" — it outputs a probability distribution over all 50,000+ possible tokens. Paris might have probability 0.74, Lyon 0.04, the 0.02, and so on. To get an actual word, we have to sample from this distribution.

Several knobs control the sampling:

ParameterWhat it doesTypical use
Temperature0 = always pick the highest-probability token (deterministic). Higher = flatter distribution, more randomness.0.0–0.3 for factual tasks, 0.7–1.0 for creative writing.
Top-p (nucleus)Only sample from the smallest set of tokens whose probabilities sum to p. Cuts off the long tail of unlikely tokens.0.9 is a common default.
Top-kOnly consider the top k most likely tokens.40 is a common default.
Frequency / presence penaltyDiscourages repeating tokens. Useful for long outputs.0.0–0.5 for varied prose.
★ Practical tip

If you ever want the same exact answer from the same prompt: set temperature to 0. If you want creative variation: increase it. If your model's giving repetitive output: try temperature 0.7 + a frequency penalty.

The Context Window: Working Memory

The context window is how much text the model can "see" at once — both your prompt and the response so far. It's measured in tokens (~4 chars each, on average).

The original GPT-3 had a 2,048-token context (~1,500 words). Modern frontier models have 128K (Claude, GPT-4 Turbo), 1M (Gemini 1.5 Pro), even 2M (Gemini 1.5 Pro extended). Context window size determines whether you can dump a whole book into a prompt or have to summarize.

⚠ Watch out

Big context window doesn't mean perfect recall. Models often perform worse on info buried in the middle of a long context (the "lost in the middle" problem). Putting key information at the start or the end works better.

Two important properties of the context window:

  • Compute scales quadratically with context length. Doubling the context costs ~4× more compute (because every token attends to every other token). This is why long contexts are expensive.
  • The model has no memory beyond context. Each conversation starts fresh. "Remembering" previous chats is a UI feature, not a model feature — the previous conversation gets stuffed into the current context.

What LLMs Are (And Aren't)

What LLMs are
  • Pattern-matchers over text. Astonishingly good ones.
  • Compressors of human knowledge — a 70B-parameter model has compressed much of public web into 140GB of weights.
  • Probabilistic. They don't have one fixed answer, they have distributions.
  • Stateless. Every call starts fresh from the context you provide.
What LLMs aren't
  • Databases. They don't store facts cleanly — facts are smeared across weights and easy to confuse with similar-sounding falsehoods.
  • Calculators. They can do math by reasoning, but they're not reliable arithmetic engines without tool use.
  • Real-time. Their knowledge stops at the training cutoff (often 6+ months ago) unless connected to search.
  • Self-aware in any meaningful sense. They have no continuous existence — each token generation is independent.
The most useful mental model: an LLM is a brilliant but amnesiac intern who's read the entire internet and works strictly off the documents you put on their desk. Treat the context window as their workspace — and their entire memory.
💼 Pro Talk · How to sound fluent

"Autoregressive" — generating one token at a time, feeding output back in. "Decoder-only" — the GPT/Claude/Llama architecture (no encoder).

"Tokens per second (TPS)" — generation speed. Frontier APIs typically deliver 50–200 TPS.

"TTFT" (time to first token) — latency before output starts. Matters for chat UX.

"Hallucination" — when the model generates plausible-sounding but false text. Inherent to next-token prediction; mitigated by RAG, tool use, careful prompting.

"Quantization" — running the model with lower-precision numbers (e.g., 4-bit instead of 16-bit). Cuts memory ~4× with small quality loss. Standard for running open models on consumer hardware.

"MoE" (mixture of experts) — an architecture where only some weights activate per token. Lets a 700B-param model run at the speed of a 70B-param one. Used by Mixtral, GPT-4, Gemini, DeepSeek.

Self-Check Questions

🧠 Test yourself · Answer out loud
  1. What does "autoregressive" mean for an LLM?
    Answer

    The model generates one token at a time, with each newly produced token appended to the input for the next prediction. The output of step N becomes part of the input for step N+1.

  2. What does "temperature 0" do?
    Answer

    Makes generation deterministic — always picks the highest-probability next token. Same prompt produces the same output. Useful for factual tasks, testing, and reproducibility.

  3. What's the context window, and why does it matter?
    Answer

    The maximum amount of text (in tokens) the model can attend to in a single call. It's the model's working memory. Anything outside doesn't exist for that call.

  4. Why do LLMs hallucinate?
    Answer

    Because they're trained to predict plausible next tokens, not truth. When unsure, they generate confident-sounding text that pattern-matches what truth would look like. The architecture optimizes for plausibility, not accuracy.

  5. Why is a bigger context window not always strictly better?
    Answer

    Compute and cost scale quadratically with context length. Models also tend to lose accuracy on info buried in the middle of long contexts. Targeted retrieval (RAG) is often better than dumping everything in.
