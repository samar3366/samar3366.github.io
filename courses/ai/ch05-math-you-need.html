Kid Translation: Math as Maps

🧒 Kid Translation

Don't worry — we won't actually do the math. We just need to understand what kind of math AI uses, and what it's for.

Think of math as different kinds of maps. A regular map shows where places are. A weather map shows temperature and wind. A treasure map shows a path. AI uses four kinds of "maps":

Vectors are arrows with a direction. Matrices are tables that transform vectors. Gradients point downhill on a landscape of error. Probability describes how confident we are about each guess.

That's the entire math toolkit. We'll meet each one with pictures, not formulas.

Vectors: Numbers with Direction

A vector is just a list of numbers. [3, 5] is a 2-dimensional vector. [3, 5, 1.7] is 3-dimensional. [3, 5, 1.7, 0.2, 9, 4, 8.1, ..., 0.6] with 768 entries is a 768-dimensional vector — and that's exactly the size of an embedding from a small language model.

You can visualize a vector as an arrow from the origin. In 2D and 3D this works easily. In 768D, it doesn't — but the math is the same.

This idea — measuring meaning by angle between vectors — is the heart of how modern AI represents knowledge. We'll come back to it big-time in Chapter 17 (Embeddings).

Matrices: Tables of Numbers

A matrix is a 2D table of numbers. A 3×4 matrix has 3 rows and 4 columns. Why does AI care?

Because matrices transform vectors. Multiply a vector by a matrix, and you get a new vector — possibly in a totally different "space." That's exactly what each layer of a neural network does.

— A Layer's Whole Job, in One Equation —
output = activation( W × input + b )

Here W is a matrix of weights, input is a vector, b is a bias vector. The matrix multiplication takes the input from one space (say, 768 numbers) into another (say, 3072 numbers, which is the next layer's width). The activation adds the non-linearity. That's an entire neural network layer in one line.

Modern GPUs are absurdly fast at matrix multiplication. That's literally why they accelerated AI so much — most of what a network does, in volume, is matrix multiplies.

Gradients: Which Way Is Down?

Imagine a hilly landscape where height = "how wrong my model is." Training is just walking downhill until you find a valley.

The gradient at a point is an arrow that points uphill — in the direction of steepest increase. So if you walk in the opposite direction of the gradient, you're going downhill. That's gradient descent: the algorithm behind nearly all neural network training.

The math fact you need to know: gradients are computed by calculus (derivatives). The whole field of "automatic differentiation" exists to compute gradients automatically through neural network code. Backpropagation is a specific algorithm to do this efficiently.

Variants like SGD, Adam, AdamW, Adafactor are all just different recipes for "how big a step to take, in what direction, given the gradient." You'll hear "Adam" most often — it's a smart adaptive step-size optimizer.

Probability: Living with Uncertainty

AI almost never says "the answer is X." It says "I'm 73% sure it's X, 22% sure it's Y, 5% spread across the rest." Probability is how machines express uncertainty.

You'll hear lots of probability terms. The most useful ones:

  • Softmax: the function that turns raw scores into a probability distribution that sums to 1.
  • Cross-entropy: a loss function that punishes the model when its predicted distribution differs from the true answer. The standard loss for classification.
  • Bayesian: any reasoning that updates probabilities as evidence comes in. Comes up in some advanced AI techniques.
  • Maximum likelihood: the principle of "pick the model parameters that make the observed data most probable." Almost all neural network training is, deep down, maximum likelihood.

High Dimensions: Where AI Lives

Here's the most counterintuitive thing about modern AI: everything happens in extremely high-dimensional space. A small embedding has 768 dimensions. A modern LLM lives in spaces with thousands of dimensions.

Our brains can only visualize up to 3D. So when you read about an "embedding space," you have to do an act of imagination: pretend you're in a 768D room. Each "direction" in that room corresponds to some abstract feature the network found useful. Maybe direction #1 is "formality." Direction #2 is "subject is a person." Direction #100,003 is "things related to honey." Nobody told the network to use these — it organized them itself during training.

The tools (vectors, matrices, gradients, probabilities) are simple. The trick is they're applied in unimaginably high-dimensional space. That's where intelligence emerges.
💼 Pro Talk · How to sound fluent

"Cosine similarity between embeddings" — the angle-based similarity we just saw. Used everywhere in semantic search.

"The loss landscape is non-convex" — meaning there are many local valleys, not one big bowl. Gradient descent isn't guaranteed to find the global minimum.

"Stochastic gradient descent (SGD)" — gradient descent that uses random batches of data for each step instead of the whole dataset. Faster and surprisingly effective.

"Adam optimizer" — the most popular practical gradient-descent variant. Adapts step sizes per-parameter automatically.

"This model has 768-dimensional embeddings" — each input gets represented as a vector with 768 numbers. Higher-dim = more capacity, more compute.

Self-Check Questions

🧠 Test yourself · Answer out loud
  1. What does the angle between two vectors tell you?
    Answer

    How similar they are. Small angle = very similar. 90° = unrelated. 180° = opposite. The numerical version is "cosine similarity."

  2. What does a matrix multiplication actually do to a vector?
    Answer

    It transforms the vector — possibly into a different space (different dimensionality), possibly rotating or stretching it. Each layer of a neural network is one such transformation.

  3. What direction does the gradient point, and which direction do we walk during training?
    Answer

    The gradient points uphill (toward higher loss). We walk in the opposite direction (downhill, toward lower loss). That's why it's called gradient descent.

  4. Why does AI deal with probabilities rather than single answers?
    Answer

    Real-world data is ambiguous. Saying "73% cat, 22% dog" is more honest and more useful than just saying "cat." It also lets you measure confidence and combine evidence.

  5. If a model has 4096-dimensional embeddings, what does that mean intuitively?
    Answer

    Each input is represented as a list of 4096 numbers — a point in a 4096-dimensional space. Each dimension corresponds to some learned feature of the input.
