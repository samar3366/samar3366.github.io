Embeddings & Semantic Search

Kid Translation: A Library Where Similar Books Live Together

🧒 Kid Translation

Imagine a library — but instead of organizing books by alphabet, the librarian organizes them by meaning. All cookbooks sit together. Cat books and dog books are in the same neighborhood (animals). Mystery novels are far from physics textbooks but right next to thrillers.

Now imagine the librarian uses GPS coordinates instead of shelves: every book has an exact (x, y, z, ...) location. Books with similar topics get nearby coordinates. To find books like the one you just read, you just measure distances. The closest 5 are your top recommendations.

That's an embedding. Each book (or sentence, image, song, code snippet) gets a coordinate in a high-dimensional "meaning space." Similar things end up close. Different things end up far. Once you've put everything in this space, search and recommendation become geometry — fast, simple, powerful.

What Embeddings Actually Are

An embedding is a vector — a list of numbers — that represents a piece of content. For text, an embedding might look like [0.23, -0.41, 0.07, ..., 0.18] with 768, 1024, 1536, or 3072 dimensions depending on the model.

You produce them with an embedding model: feed in text, get out a vector. The model has been trained so that texts with similar meaning produce similar vectors. The model itself is usually a transformer encoder; the vector is taken from its final-layer output.

Concrete example

Embed three sentences with OpenAI's text-embedding-3-small (1536 dims):

  • "How do I cancel my subscription?" → vector A
  • "What's the refund policy?" → vector B
  • "How does photosynthesis work?" → vector C

A and B are near-neighbors despite sharing barely any words — the model "understood" they're both about leaving a service. C is far from both. The geometry encodes meaning, not vocabulary.

The Geometry of Meaning

Once concepts are vectors, you can do arithmetic on meaning. The classic example from word embeddings:

— Famous Word Embedding Arithmetic —
vector("king") − vector("man") + vector("woman") ≈ vector("queen")

The geometry captures relationships. Subtracting "man" from "king" pulls out a "royalty without gender" direction; adding "woman" applies female gender; the result lands near "queen." Modern embeddings are richer — these directions encode hundreds of thousands of relationships, learned from data.

Words isn't the only thing that embeds. Images, audio, code, molecules — anything you have data for can be embedded. Multimodal embeddings like CLIP put text and images in the same space, so you can search images with text queries.

Cosine Similarity: The Workhorse

To measure how close two vectors are in meaning, the standard metric is cosine similarity — the cosine of the angle between them. It ranges from −1 (opposite) through 0 (unrelated) to 1 (identical direction).

Why cosine and not Euclidean distance? Cosine ignores vector magnitude, focusing only on direction. Two embeddings can have very different lengths (e.g., a long document vs. a short query) but still point the same direction in meaning-space. Cosine captures that. Most embedding models are explicitly trained to put meaning in direction.

Vector Databases & ANN Search

Searching a million vectors by exact cosine similarity means a million dot products — fast, but not fast enough at internet scale. Approximate nearest neighbor (ANN) algorithms trade a small accuracy hit for huge speed gains.

The dominant ANN algorithm is HNSW (Hierarchical Navigable Small World) — it builds a graph of vectors connected to their neighbors at multiple layers, letting you "navigate" to the nearest match in milliseconds even with billions of vectors. Most modern vector databases use HNSW or a variant under the hood.

Beyond RAG: Other Uses

Embeddings power a lot more than RAG:

  • Semantic search — the core use case, in products like documentation search, e-commerce search, code search.
  • Clustering — group similar items (customer feedback, support tickets, news articles) without needing labels.
  • Classification — embed the text, then use a small classifier on top. Often beats fine-tuning for low-data scenarios.
  • Deduplication — find near-duplicates that have different exact wording.
  • Recommendation — embed users and items in the same space; nearest neighbors are recommendations.
  • Anomaly detection — items far from any cluster are anomalies.
  • Image / audio search — same idea, different modalities. CLIP for images, CLAP for audio.
Embeddings turn meaning into geometry. Once you have meaning as coordinates, all the tools of geometry — distance, clustering, projection — become tools for understanding data.
💼 Pro Talk · How to sound fluent

"Dense embeddings" — every dimension has a value (the standard). "Sparse embeddings" — most dimensions are zero (e.g., BM25 vectors).

"ANN search" — approximate nearest neighbor. "HNSW" — the dominant graph-based ANN algorithm.

"MTEB" — the embedding benchmark. "BEIR" — the retrieval benchmark.

"Matryoshka embeddings" — embeddings designed so you can truncate to a smaller dim and still get usable similarity. Cuts storage costs.

"Bi-encoder vs cross-encoder" — bi-encoder embeds query and doc separately (fast, retrieval). Cross-encoder embeds them together (slower, more accurate, used for reranking).

"ColBERT-style" — a multi-vector embedding approach where each token gets its own vector; often more accurate than single-vector embeddings.

"Quantization" — store vectors in 8-bit or 4-bit to cut memory ~4–8× with minor accuracy loss.

Self-Check Questions

🧠 Test yourself · Answer out loud
  1. What is an embedding, in one sentence?
    Answer

    A vector of numbers that represents a piece of content (text, image, etc.) such that semantically similar content gets vectors with similar direction.

  2. Why use cosine similarity instead of Euclidean distance?
    Answer

    Cosine ignores magnitude and focuses on direction, which matches how embedding models encode meaning. Two texts of very different length but the same topic should still rate as similar — Euclidean distance would penalize the length difference.

  3. What does ANN ("approximate nearest neighbor") buy you over exact search?
    Answer

    Vastly faster query times at scale (milliseconds for billions of vectors) at the cost of a small accuracy drop. Critical for production semantic search.

  4. When should you reach for pgvector vs. a specialized vector DB?
    Answer

    Use pgvector when your data is small-to-medium (under ~10M vectors) and you already have Postgres. Use specialized DBs when you need extreme scale, advanced features (hybrid search, multi-vector), or fully managed cloud ops.

  5. Name three uses of embeddings besides RAG.
    Answer

    Many valid: semantic search, clustering, classification, deduplication, recommendation, anomaly detection, image/audio search.
